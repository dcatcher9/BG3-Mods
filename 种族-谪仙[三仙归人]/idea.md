# 谪仙 — 五行统一框架设计思路

---

# 第一部分：基础规则

## 设计原则

> 旧系统仅作灵感参考，新系统以五行 Walk/Graph 框架为准。
> 所有机制从同一套规则自然涌现，不追求与旧版一一对应。

## 核心哲学

> 五行既是名词（灵根），也是动词（式）。
> 所有技能、功法，都从同一套规则涌现，不需要手写每一种组合。

---

## 五行定义与核心公式

| 元素 | index | 灵根（体/静） | 式（用/动） |
|------|-------|-------------|------------|
| 木 | 0 | 生命力、体质 | 生：治疗、生长、恢复 |
| 火 | 1 | 攻击力、破坏 | 伤：造成伤害、燃烧 |
| 土 | 2 | 防御力、稳固 | 御：承受、格挡 |
| 金 | 3 | 精准度、锋利 | 断：切割、精确打击 |
| 水 | 4 | 灵活度、流动 | 化：变化、适应、转化 |

```
distance = (to_index - from_index + 5) % 5

distance 0 = 共鸣（同元素）
distance 1 = 生（协同）
distance 2 = 克（制约）
distance 3 = 侮（反叛）
distance 4 = 泄（偏转）

reaction_tier = min(tier_A, tier_B)
```

**方向决定效果：**
- 木→火：distance=(1-0+5)%5 = 1 = 生（木赋能火）
- 火→木：distance=(0-1+5)%5 = 4 = 泄（火消耗木）

**相生验证：** 木→火→土→金→水→木，每步 distance=1 ✓
**相克验证：** 木→土→水→火→金→木，每步 distance=2 ✓

## 灵根系统

灵根 = 脏器节点的 tier 值。

| Tier | 阶称 | 灵根值门槛 |
|------|------|----------|
| T0 | 凡品 | ≥ 25 |
| T1 | 天品 | ≥ 100 |
| T2 | 仙品 | ≥ 300 |
| T3 | 圣品 | ≥ 1000 |

**双组合 = 走对应经脉那条有向边时触发的命名效果：**

```
开脉门槛：两端灵根均 ≥ 50
触发条件：经脉已开通 + walk 经过这条有向边
效果强度：reaction_tier = min(tier_A, tier_B)
```

夺灵偷到弱灵根 → 经脉仍开着，但 min() 被拉低 → 效果变弱，自然惩罚。
三灵根以上不需要手写：多条经脉同时开通 → 同一 walk 里多个命名效果自然叠加。

**5种行为经验**（用于开脉条件和功法结晶）：
- 木之行：治疗/恢复 (OnHeal)
- 火之行：伤害/击杀 (OnDamage, IsKillingBlow)
- 土之行：承伤/格挡 (OnAttacked, 未移动回合)
- 金之行：暴击/反击 (IsCritical, 使用反应)
- 水之行：偷取/变形 (Steal status, Polymorph)

## 经脉系统：身体就是图

### 节点（6个，固定）

**五脏（有元素）+ 丹田（中性枢纽）**：

```
           丹田（中性）
          /  |  |  |  \
         木  火  土  金  水
        肝  心  脾  肺  肾
          \___|__|__|__/
             （相互之间还有10条经脉）
```

| 节点 | 元素 | index |
|------|------|-------|
| 肝 | 木 | 0 |
| 心 | 火 | 1 |
| 脾 | 土 | 2 |
| 肺 | 金 | 3 |
| 肾 | 水 | 4 |
| **丹田** | **中性** | — |

### 经脉 = 有向边（双向，15条，开/关）

经脉是双向有向边（A⇄B = A→B 且 B→A）。
15条 = 10条脏器间 + 5条丹田经脉。

**脏器间经脉（10条）：**

```
经脉       A→B distance   B→A distance
────────────────────────────────────
木─火         1=生           4=泄
火─土         1=生           4=泄
土─金         1=生           4=泄
金─水         1=生           4=泄
水─木         1=生           4=泄
────────────────────────────────────
木─土         2=克           3=侮
土─水         2=克           3=侮
水─火         2=克           3=侮
火─金         2=克           3=侮
金─木         2=克           3=侮
────────────────────────────────────
```

**丹田经脉（5条）：**

```
脏器 → 丹田 = 归元（蓄积、自身强化）
丹田 → 脏器 = 发元（释放、增幅该元素效果）
```

15条 × 双向 = **30条有向边**，每条开通即解锁一个技能方向。

### 丹田经脉的特殊作用

```
路过丹田 = 绕过元素克制，直通本源

木 → 丹田 → 金：
  不经过 金克木 的对抗关系
  → 生命力直接增幅精准度

含丹田的环（≥3节点）= 最强被动：
  木 → 火 → 丹田 → 木
  = 打击越强，生命越旺
```

### 开脉条件

```
开通脏器间经脉 X─Y：
  1. 灵根_X ≥ 50
  2. 灵根_Y ≥ 50
  3. 行为_X 或 行为_Y 积累达标

开通丹田经脉 丹田─X：
  1. 灵根_X ≥ 高阶门槛（比脏器间更难）
  2. 任意行为积累达到一定总量
```

## 气的流向规则

```
1. 主动走（有神识引导）：玩家/功法指定方向
2. 自流（无神识）：
   - 从 tier 较高的节点流向 tier 较低的节点
```

方向决定 distance：同一条经脉 A─B，走 A→B 和走 B→A 效果完全不同。

# 第二部分：Walk 系统

## Walk 模型：技能 = 游走序列

技能不是图的静态拓扑，而是一段**气在节点间的游走（Walk）**。
节点和经脉可以重复经过。

```
例：[木→火→土→火→土→金]
  节点 火、土 各经过两次
  经脉 火─土 经过两次（两个方向）
```

底层图（15条经脉的开/关）= 定义哪些步骤合法
Walk 序列 = 技能的实际定义

---

## Walk 效果计算

### 层1：即时效果（每条边触发）

每走一步，触发该边的反应：

```
Walk: 木→火→土→火→土→金
  木→火: d=1=生 → 即时效果①
  火→土: d=1=生 → 即时效果②
  土→火: d=4=泄 → 即时效果③
  火→土: d=1=生 → 即时效果②（叠加）
  土→金: d=1=生 → 即时效果④
```

### 层2：节点转化效果（Markov）

节点记住气是怎么到达的，影响气离开时的性质：

```
node_compound = (d_in + d_out) % 5

中间节点 火：入边=生(1)，出边=生(1) → compound=(1+1)%5=2=克
中间节点 土：入边=生(1)，出边=泄(4) → compound=(1+4)%5=0=共鸣
```

### 层3：最终效果

```
final_distance = (Σ 所有边的 distance) % 5
```

```
Walk: 木→火→土→火→土→金
distances: 1+1+4+1+1 = 8, 8%5 = 3 = 侮
→ Walk 结束时触发一次强力「侮」效果
```

**分形自洽：同一公式在所有层面成立**

| 层面 | 计算 |
|------|------|
| 单条边 | distance(A→B) |
| 中间节点 | (d_in + d_out) % 5 |
| 整段 Walk | (Σ all d) % 5 |
| 图-图复合 | distance(output→input) |

**推论：大周天恒为共鸣**
```
相生大周天：1+1+1+1+1=5, 5%5=0=共鸣
相克大周天：2+2+2+2+2=10, 10%5=0=共鸣
任何完整五行大周天，最终效果恒为共鸣(0)。
```

## 技能分类（单段 Walk）

### 主动 vs 被动

```
Walk 有终点，不循环   → 主动
Walk 包含有向环(≥3)  → 被动
```

A→B 且 B→A 是一条双向经脉，**不算被动环**（长度=2不触发）。

### 主动技能分类

| Walk 结构 | 类型 | BG3形式 |
|----------|------|---------|
| 2节点链 A→B | 即时技能 | Bonus Action |
| 3节点链 A→B→C | 技能/连招 | Action |
| 4节点链+ | 大招 | Action + 前置条件 |
| 有分支（树）A→{B,C} | 法术/AoE | Spell slot |
| 多汇聚 {A,B}→C | 蓄力/仪式 | 需要预备动作 |

效果指向由**末端节点元素**决定：

| 末端元素 | 效果指向 |
|---------|---------|
| 木、土 | 自身（治疗/防御）|
| 火、金 | 敌方（伤害/削甲）|
| 水 | 范围（位移/控制）|

### 被动技能分类

| 环的形状 | 节点数 | 类型 |
|---------|--------|------|
| 三角环 | 3 | 脉冲被动（每回合触发）|
| 四节点环 | 4 | 强脉冲被动 |
| 小周天（相生序环）| 3 | 稳定 aura |
| 大周天（5节点完整相生）| 5 | 领域被动 |
| 全连通 | 5 | 混沌领域 |

**所有被动的 final 恒为共鸣(0)。** 效果由每步边的即时效果叠加体现。

被动解锁门槛：需要至少3条经脉形成有向环。

```
开脉 木─火 + 火─土 + 木─土
→ 三脉成环 木→火→土→木 = 第一个被动！
```

# 第三部分：图的层级

## 复合技能（图-图 Walk）

图本身可以作为更高层图的节点（递归组合）。

```
图 = { input: 触发元素, output: 产出元素, body: 内部结构 }
连接规则：A.output → B.input，distance 用同一公式计算
```

```
例：
  修罗环（木→火→土→木，output: 火）
  血刃链（火→金，input: 火）

  修罗环 → 血刃链：distance(火→火)=0=共鸣
  → 两个 walk 同时共存，效果自然叠加
```

---

## 递归组合（无限层级）

```
招式      = Level 1 图  （单段 walk，战斗即时技能）
复合招式  = Level 2 图  （多段 walk 的图-图组合）
功法      = Level 3 图  （改变 walk 物理规则的修炼体系）
天道      = Level N 图  （接近宇宙规律本身）
```

**量变到质变的机制 = 递归组合本身。**

```

```

```
变化：丹田激活（中性枢纽）
新增：5条丹田经脉可开（归元/发元）

```

### 金丹期 ★关键选择★
```
变化：丹田（中性）→ 金丹（固定元素，不可逆）
      金丹元素 = 突破时主灵根元素

图变化：丹田经脉从归元/发元 → 真正的 distance 公式
        15条经脉全部平等，Wu Xing 覆盖全图

火金丹示例：
  丹田→心(火): distance=0=共鸣（最强增幅）
  丹田→肝(木): 泄（火消耗木）
节点数：6（全图 distance 统一，质变）
```

### 元婴期
```
变化：金丹内孕育元婴，独立意识节点
      元婴可有不同于金丹的元素

新增能力：元婴可独立执行并行 walk

  → 两个 final 效果同时触发

```

### 化神期

```
变化：灵与肉分离，图获得两个并行层
物质层：5脏器 + 金丹
精神层：5脏器镜像 + 元婴

Walk 可在两层间跳跃：
  肝(身)→心(身)→心(神)→肾(神)→肾(身)
节点数：12
```

### 炼虚期
```
变化：意识外投，walk 可经过临时外部节点
外部节点：目标敌人（弱点元素）/ 环境元素 / 友方脏器

Walk 穿透目标：
  肝(木) → [敌人(土)] → 心(火) → ...

```

### 合体期
```
变化：天人合一，天地节点可接入
条件：特定环境（室外/地形/天气）
经过天地节点的 walk = 借助天地之力 = 效果大幅增强
```

### 大乘期
```
变化：召唤法相 = 在外部实例化内部图的副本
法相图 ↔ 本体图：通过元素接口连接
Walk 可从本体进入法相再回本体
= Layer 2（图-图）在游戏中的实体化
```

### 渡劫期
```
变化：天道有自己的五行图，天劫 = 天道向你发动的 walk
渡劫成功：吸收天道 walk 片段 → 获得天道经脉
```

### 真仙期
```

  distance(空→任何元素) = 玩家自选
  distance(任何元素→空) = 0（共鸣）
```

## 功法：气的运行方式

```
Walk  = 路线（经过哪些节点）     ← 玩家自己可配置
功法  = 气的运行方式（物理规律） ← 功法独有，无法自配
```

同一条 walk，有无功法效果完全不同：

```
Walk: [肝→心→脾]，无功法：
  木→火=生, 火→土=生, final=(1+1)%5=2=克

同一 walk，修「玄火诀」（火节点出边+1）：
  火→土变成克(2), final=(1+2)%5=3=侮
  完全不同的结果
```

### 功法能改变什么

| 功法能改变 | 自配 walk 能做到 |
|-----------|----------------|
| 节点的 distance 偏移 | ✗ |
| tier 折叠方式（min→max）| ✗ |
| 节点经过时的额外触发 | ✗ |
| final_distance 折叠方式 | ✗ |
| 解锁特殊经脉 | ✗ |
| walk 路线选择 | ✓ |

### 功法结构定义

```lua
功法 = {
    node_modifiers = {
        ["心(火)"] = { distance_offset = 1 },
        ["丹田"]   = { tier_boost = 1 }
    },
    edge_modifiers = {
        ["木─火"] = { extra_trigger = "水归元" }
    },
    fold_rule = "max_not_min",
    unlock    = ["特殊经脉甲"]
}
```

### 如何获得功法

**外得（快）：**

| 来源 | 方式 |
|------|------|
| 玉简/典籍 | 地牢、宝箱、NPC |
| 师传 | 加入宗门或拜师 |
| 夺取 | 击败敌人 |
| 购买 | 坊市交易 |

外得功法与金丹元素不匹配 → 效果减半；完全匹配 → 效果×1.5

**内悟（慢，完全贴合）：**

```
某条 walk 执行 ≥ N 次
+ final_distance 结果一致性 ≥ X%
+ 该 walk 的反应 tier 达到门槛
→ 功法自动从你的身体中结晶
```

内悟功法无法转让，刻在体内。

### 核心循环

```
走基础 walk → 反复使用好用的路线 → 行为经验积累
→ 内悟功法结晶 → 该 walk 效果质变
→ 外获玉简解锁新节点修正
→ walk → 功法 → 更强的 walk（正向反馈）
```

## 法宝：外部子图

法宝 = 接入体内图的外部节点集合。
炼化 = 在脏器节点与法宝节点之间开通经脉。

### 法宝层级

```
简单法宝（单节点）：
  飞剑 = 金(T2) 节点
  炼化：肺(金)─飞剑(金), distance=0=共鸣
  Walk 经过飞剑 = 共鸣增幅，飞剑同步攻击

强力法宝（多节点）：
  混元鼎 = 土(T3)─火(T2)─土(T3) 内部图
  Walk 可深入鼎内触发内部反应

至宝（完整子图）：
  内部有独立的5行节点 + 经脉
  Walk 在两图之间自由穿越
```

### 炼化过程

```
获得法宝 → 悬空节点（无经脉连接）
初步炼化 → 开通一条脏器─法宝经脉
深度炼化 → 开通更多经脉，进入法宝内部
完全炼化 → 法宝内外全通，walk 自由穿越
```

### 功法 × 法宝

```
功法 = 走路的物理规律（改变 distance/tier 计算）
法宝 = 走路时可经过的外部节点（扩展图的边界）

功法节点修正 + 法宝外部节点 + walk 路线选择
→ 三者叠加，同一条经脉走出完全不同的效果
```

## 神识：Walk 的执行者

```
灵根 = 图的节点强度
经脉 = 图的连通性
神识 = Walk 的执行精度

无神识：气被动自流，效果弱
有神识：意识主动引导，效果强
```

### 神识 = 元婴节点的 Tier

```
炼气/筑基：无元婴，神识=0，walk 只能自动流动
金丹期：金丹产生初步意识，神识=金丹 tier
元婴期：元婴节点诞生，神识强度 = 元婴 tier
化神期+：元婴可脱体外投
```

### 神识影响 Walk 的两个维度

```
1. Walk 深度上限：
   T0 → 最多 3 节点
   T1 → 最多 5 节点
   T2 → 最多 8 节点
   T3 → 无上限

2. Tier 补偿：
   reaction_tier = min(节点tiers) + 神识bonus
   神识强 → 弱节点被补偿
```

### 三个神识能力

**内观：** 元婴在自身图内完整游走一遍 → 感知所有节点状态

**洞观：** 元婴短暂外投 → 读取附近角色的图结构

**掌日：** 元婴进入目标图，强制执行 walk → 傀儡
- 对方抵抗 = 对方元婴 tier
- 你的神识 > 对方神识 → 傀儡成功

## 天然精华（天地种）

天地种植入脏器节点，永久改变节点品质。每个节点只能种一株，可替换。

```
层A：节点强化（静态）
  tier_boost：节点 tier 永久 +N
  distance_mod：出边或入边 distance 偏移

层B：walk 触发（动态）
  extra_trigger：walk 经过此节点时额外触发
  fold_rule：改变 reaction_tier 计算方式
  special：独特机制
```

**品级：**
```
T1：tier+1 + 小触发，无境界门槛
T2：tier+2 + walk 修正，需境界 ≥ 金丹
T3：tier+3 + 强力机制，需境界 ≥ 元婴，通常唯一
```

## 阵法：多人空间图

阵法 = 空间化的多人图，参与者是节点，位置决定边。

### 八卦 = 五行的空间表达

```
坎→水  离→火  震→木  巽→木
乾→金  兑→金  艮→土  坤→土
```

8个阵位 = 8个节点，边由相邻阵位的元素决定。

### 阵法运转 = Walk 流过整个阵

```
你的内部图 ─ 你所在阵位 ─ 阵法图 ─ 其他参与者

final_distance = (Σ 所有步) % 5 = 阵法整体效果
```

| 阵法图形 | Walk 类型 | 效果 |
|---------|----------|------|
| 线型排列 | 链 | 主动阵（一次性爆发）|
| 环型排列 | 环 | 守阵（持续被动）|
| 星型 | 树 | 法阵（中心施法）|
| 完整八卦 | 完全图 | 大阵（领域效果）|

聚灵阵 = 完整八卦阵 = final=0=共鸣 → 吸引天地灵气（天地节点临时接入）

## 实现方向

```lua
ELEM_INDEX = { ["木"]=0, ["火"]=1, ["土"]=2, ["金"]=3, ["水"]=4 }

-- 经脉状态（15条，开/关）
PersistentVars['JINGMAI'] = {
    ["木火"]=false, ["火土"]=false, ["土金"]=false,
    ["金水"]=false, ["水木"]=false,
    ["木土"]=false, ["土水"]=false, ["水火"]=false,
    ["火金"]=false, ["金木"]=false,
    ["丹田木"]=false, ["丹田火"]=false, ["丹田土"]=false,
    ["丹田金"]=false, ["丹田水"]=false,
}

function EdgeDistance(from, to)
    if from == "丹田" then return "发元" end
    if to   == "丹田" then return "归元" end
    return (ELEM_INDEX[to] - ELEM_INDEX[from] + 5) % 5
end

function ClassifyRoute(nodes, edges)
    if HasDirectedCycle(nodes, edges) then
        return "PASSIVE"
    elseif HasBranching(edges) then
        return "SPELL"
    else
        local n = #nodes
        if n == 2 then return "ACTIVE_BONUS"
        elseif n == 3 then return "ACTIVE_ACTION"
        else return "ACTIVE_ULTIMATE" end
    end
end

function CalcWalkEffects(walk)
    local edges = {}
    local dist_sum = 0
    for i = 1, #walk - 1 do
        local d = EdgeDistance(walk[i], walk[i+1])
        local tier = math.min(GetTier(walk[i]), GetTier(walk[i+1]))
        edges[i] = { dist=d, tier=tier, effect=REACTION_TABLE[d][tier] }
        if type(d) == "number" then dist_sum = dist_sum + d end
    end
    local node_effects = {}
    for i = 2, #walk - 1 do
        local d_in  = edges[i-1].dist
        local d_out = edges[i].dist
        if type(d_in) == "number" and type(d_out) == "number" then
            local compound = (d_in + d_out) % 5
            node_effects[i] = { dist=compound, effect=REACTION_TABLE[compound][edges[i].tier] }
        end
    end
    local final_dist = dist_sum % 5
    local tiers = {}
    for _, e in ipairs(edges) do tiers[#tiers+1] = e.tier end
    local final_tier = math.min(table.unpack(tiers))
    return {
        edge_effects  = edges,
        node_effects  = node_effects,
        final_dist    = final_dist,
        final_effect  = REACTION_TABLE[final_dist][final_tier],
    }
end
```
