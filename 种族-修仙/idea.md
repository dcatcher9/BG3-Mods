# 第一部分：基础规则

## 设计原则

> 所有机制从同一套规则自然涌现，不追求手写每一种组合。
> 五行既是名词（灵根），也是动词（式）。

---

## 五行定义与核心公式

| 元素 | index | 灵根（体/静） | 式（用/动） |
|------|-------|-------------|------------|
| 木 | 0 | 生命力、体质 | 生：治疗、生长、恢复 |
| 火 | 1 | 攻击力、破坏 | 伤：造成伤害、燃烧 |
| 土 | 2 | 防御力、稳固 | 御：承受、格挡 |
| 金 | 3 | 精准度、锋利 | 断：切割、精确打击 |
| 水 | 4 | 灵活度、流动 | 化：变化、适应、转化 |

```
distance = (to_index - from_index + 5) % 5

distance 0 = 共鸣（同元素）
distance 1 = 生（协同）
distance 2 = 克（制约）
distance 3 = 侮（反叛）
distance 4 = 泄（偏转）

reaction_tier = min(tier_A, tier_B)
```

**方向决定效果：**
- 木→火：distance=(1-0+5)%5 = 1 = 生（木赋能火）
- 火→木：distance=(0-1+5)%5 = 4 = 泄（火消耗木）

**相生验证：** 木→火→土→金→水→木，每步 distance=1 ✓
**相克验证：** 木→土→水→火→金→木，每步 distance=2 ✓

## 灵根系统

灵根 = 脏器节点的 tier 值。

| Tier | 阶称 | 灵根值门槛 |
|------|------|----------|
| T0 | 凡品 | ≥ 25 |
| T1 | 天品 | ≥ 100 |
| T2 | 仙品 | ≥ 300 |
| T3 | 圣品 | ≥ 1000 |

**双组合 = 走对应经脉那条有向边时触发的命名效果：**

```
开脉门槛：两端灵根均 ≥ 50
触发条件：经脉已开通 + walk 经过这条有向边
效果强度：reaction_tier = min(tier_A, tier_B)
```

夺灵偷到弱灵根 → 经脉仍开着，但 min() 被拉低 → 效果变弱，自然惩罚。
三灵根以上不需要手写：多条经脉同时开通 → 同一 walk 里多个命名效果自然叠加。

## 共鸣被动（自环 Walk）

> 当灵根 > 0 时，该脏器节点解锁共鸣被动（自环 A→A）。
> 随着灵根 Tier 提升，被动效果自动升级。
> 它是最简单的 Walk，也是修行的起点 — 脏器与自身元素谐振。
>
> 自环 Walk 的 cost 和 growth 公式与多节点 Walk 完全统一：
>   qi_cost = min(tier, tier) + 1 = tier + 1（最低1气/回合）
>   shenshi_cost = distance = 0（共鸣无需引导）
>   growth = BASE_LG × qi_cost（仅该元素获得成长）

### 引导循环

```
初始灵根 → 灵根>0 → 最弱共鸣被动激活
→ 每回合自环Walk执行 → 该元素灵根缓慢成长
→ 灵根达到T0(25) → 共鸣被动升级，效果变强
→ 两个元素达到50 → 第一条经脉开通 → 第一个双节点Walk
→ Walk驱动成长接管 → 更多经脉 → 更强Walk → 正反馈
→ 灵根达到T1/T2/T3 → 共鸣被动持续升级
```

## 边效果表（20条有向边 × 3 Tier + 共鸣）

> **设计原则：**
> - **边 = 动词**（做什么）— 20种边效果，全局共享，任何 walk 经过同一条边触发同样类型的效果
> - **tier = 副词**（做到什么程度）— 高 tier 不只是数字放大，必须质变
> - **扩散/感染专责「滋」边** — 其他边不做扩散，传播行为只从水→木来
> - **重复覆盖优先于唯一性** — 多条边可以有相似效果（如减AC），全覆盖比无重叠更重要

### 异元素推演：修改 distance 影响 Layer 2/3

> 异元素（天然精华）的 distance 偏移只影响 Layer 2/3 复合效果。
> Layer 1 边效果（燃/锻/铸...）由 (from, to) 元素对固定，不受 distance 偏移改变。

```
三昧真火（火出边 distance +1）：
  Layer 1：燃还是燃，锻还是锻（不变）
  Layer 2：compound = (d_in + d_out_modified) % 5 → 节点复合效果改变
  Layer 3：Σd_modified % 5 → final 效果改变
  → 异元素改变 walk 的整体"化学性质"，不改变单条边的"物理性质"
```

## 节点（6个，固定）

**五脏（有元素）+ 丹田（中性枢纽）**：

```
           丹田（中性）
          /  |  |  |  \
         木  火  土  金  水
        肝  心  脾  肺  肾
          \___|__|__|__/
             （相互之间还有10条经脉）
```

| 节点 | 元素 | index |
|------|------|-------|
| 肝 | 木 | 0 |
| 心 | 火 | 1 |
| 脾 | 土 | 2 |
| 肺 | 金 | 3 |
| 肾 | 水 | 4 |
| **丹田** | **中性** | — |

## 经脉 = 有向边（双向，15条，开/关）

经脉是双向有向边（A⇄B = A→B 且 B→A）。
15条 = 10条脏器间 + 5条丹田经脉。

**脏器间经脉（10条）：**

```
经脉       A→B distance   B→A distance
────────────────────────────────────
木─火         1=生           4=泄
火─土         1=生           4=泄
土─金         1=生           4=泄
金─水         1=生           4=泄
水─木         1=生           4=泄
────────────────────────────────────
木─土         2=克           3=侮
土─水         2=克           3=侮
水─火         2=克           3=侮
火─金         2=克           3=侮
金─木         2=克           3=侮
────────────────────────────────────
```

**丹田经脉（5条）：**

```
脏器 → 丹田 = 归元（蓄积、自身强化）
丹田 → 脏器 = 发元（释放、增幅该元素效果）
```

15条 × 双向 = **30条有向边**，每条开通即解锁一个技能方向。

## 丹田经脉的特殊作用

```
路过丹田 = 绕过元素克制，直通本源

木 → 丹田 → 金：
  不经过 金克木 的对抗关系
  → 生命力直接增幅精准度

含丹田的环（≥3节点）= 最强被动：
  木 → 火 → 丹田 → 木
  = 打击越强，生命越旺
```

## 开脉条件

```
开通脏器间经脉 X─Y：
  1. 灵根_X ≥ 50
  2. 灵根_Y ≥ 50

开通丹田经脉 丹田─X：
  1. 灵根_X ≥ 高阶门槛（比脏器间更难）
  2. 灵根总值达到一定总量
```

# 第三部分：Walk 系统

## Walk 模型：技能 = 游走序列

技能不是图的静态拓扑，而是一段**气在节点间的游走（Walk）**。
节点和经脉可以重复经过。

```
例：[木→火→土→火→土→金]
  节点 火、土 各经过两次
  经脉 火─土 经过两次（两个方向）
```

底层图（15条经脉的开/关）= 定义哪些步骤合法
Walk 序列 = 技能的实际定义

## Walk 效果计算

### 层1：即时效果（每条边触发）

每走一步，触发该边的反应：

```
Walk: 木→火→土→火→土→金
  木→火: d=1=生 → 即时效果①
  火→土: d=1=生 → 即时效果②
  土→火: d=4=泄 → 即时效果③
  火→土: d=1=生 → 即时效果②（叠加）
  土→金: d=1=生 → 即时效果④
```

### 层2：节点转化效果（Markov）

节点记住气是怎么到达的，影响气离开时的性质：

```
node_compound = (d_in + d_out) % 5

中间节点 火：入边=生(1)，出边=生(1) → compound=(1+1)%5=2=克
中间节点 土：入边=生(1)，出边=泄(4) → compound=(1+4)%5=0=共鸣
```

### 层3：最终效果

```
final_distance = (Σ 所有边的 distance) % 5
```

```
Walk: 木→火→土→火→土→金
distances: 1+1+4+1+1 = 8, 8%5 = 3 = 侮
→ Walk 结束时触发一次强力「侮」效果
```

**分形自洽：同一公式在所有层面成立**

| 层面 | 计算 |
|------|------|
| 单条边 | distance(A→B) |
| 中间节点 | (d_in + d_out) % 5 |
| 整段 Walk | (Σ all d) % 5 |
| 图-图复合 | distance(output→input) |

**推论：所有环的 final 恒为共鸣(0)**

```
数学证明：环 A→B→C→A
  total = (B-A) + (C-B) + (A-C) = 0 mod 5

相生大周天：1+1+1+1+1=5, 5%5=0=共鸣 ✓
相克大周天：2+2+2+2+2=10, 10%5=0=共鸣 ✓
任何环，无论大小和路径，final 恒为共鸣(0)。
```

## 气的流向规则

```
1. 主动走（有神识引导）：玩家/功法指定方向
2. 自流（无神识）：
   - 从 tier 较高的节点流向 tier 较低的节点
   - tier 相等 → 相生序优先（index 小的先流出）
```

方向决定 distance：同一条经脉 A─B，走 A→B 和走 B→A 效果完全不同。

## 技能分类（单段 Walk）

### 主动 vs 被动

```
Walk 有终点，不循环   → 主动
Walk 包含有向环(≥3)  → 被动
自环 A→A             → 被动（最简被动）
```

A→B 且 B→A 是一条双向经脉，**不算被动环**（长度=2不触发）。

### 主动技能分类

| Walk 结构 | 类型 | BG3形式 |
|----------|------|---------|
| 2节点链 A→B | 即时技能 | Bonus Action |
| 3节点链 A→B→C | 技能/连招 | Action |
| 4节点链+ | 大招 | Action + 前置条件 |
| 有分支（树）A→{B,C} | 法术/AoE | Spell slot |
| 多汇聚 {A,B}→C | 蓄力/仪式 | 需要预备动作 |

效果指向由**末端节点元素**决定：

| 末端元素 | 效果指向 |
|---------|---------|
| 木、土 | 自身（治疗/防御）|
| 火、金 | 敌方（伤害/削甲）|
| 水 | 范围（位移/控制）|

### 被动技能分类

| 环的形状 | 节点数 | 类型 |
|---------|--------|------|
| 自环 A→A | 1 | 共鸣被动（灵根>0解锁，0神识/回合） |
| 三角环 | 3 | 脉冲被动（每回合触发）|
| 四节点环 | 4 | 强脉冲被动 |
| 小周天（相生序环）| 3 | 稳定 aura |
| 大周天（5节点完整相生）| 5 | 领域被动 |
| 全连通 | 5 | 混沌领域 |

**所有被动的 final 恒为共鸣(0)。** 效果由每步边的即时效果叠加体现。

被动解锁门槛：需要至少3条经脉形成有向环（自环除外）。

```
开脉 木─火 + 火─土 + 木─土
→ 三脉成环 木→火→土→木 = 第一个被动！
```

# 第四部分：Cost 系统

## 核心规则

**统一规则：被动与主动使用相同的 cost 公式。**
区别仅在于：主动一次性消耗，被动每回合消耗。
当气或神识不足时，被动自动停止运转。

### 每条边的消耗

```
神识消耗/边 = distance 值
  d=0 共鸣：0（同频，无需引导）
  d=1 生：  1（顺流，自然方向）
  d=2 克：  2（压制对方，中等消耗）
  d=3 侮：  3（以弱胜强，高消耗）
  d=4 泄：  4（完全逆流，最高消耗）

气消耗/边 = min(tier_A, tier_B) + 1
  （+1 保证每条边至少消耗1气，即使两端都是T0）
```

### 主动技能 cost（一次性消耗）

```
神识 = Σ(每条边的 distance)
气   = Σ(每条边的 min(tier_A, tier_B) + 1)
HP   = 侮(d=3)边的代价比例（按代价:增益比计算）
```

### 被动技能 cost（每回合消耗）

```
神识/回合 = Σ(每条边的 distance)
气/回合   = Σ(每条边的 min(tier_A, tier_B) + 1)

气：长休恢复 → 被动每天能运转多少回合取决于丹田容量
神识：短休恢复 → 可以中途恢复继续运转
```

### 冷却

```
含 T3 边效果的 walk → 触发后有冷却回合数
被动环含 T3 边 → 每 N 回合触发一次 T3 效果（其他层级正常运转）
```

### 四种资源总览

| 资源 | 消耗来源 | 主动技能 | 被动（每回合） |
|------|---------|---------|-------------|
| 神识 | 引导气流方向 | 消耗 Σ(distance) | 消耗 Σ(distance) |
| 气 | 边效果的能量 | 消耗 Σ(min(tier)+1) | 消耗 Σ(min(tier)+1) |
| HP | 侮型(d=3)代价 | 按代价:增益比 | 按代价:增益比/回合 |
| 冷却 | T3边效果 | 触发后冷却 | 每N回合触发 |

### 推演验证

```
自环 木(T0)→木 被动：
  神识/回合 = 0（distance=0）
  气/回合   = 1（min(0,0)+1=1）
  → 丹田=2 时可运转2回合/长休

3-node cycle 木→火→土→木 (all T0)：
  神识/回合 = 1+1+3 = 5
  气/回合   = 1+1+1 = 3
  → 丹田=2, 识海=1 时无法维持（太贵）
  → 金丹期(丹田=15, 识海=3) 可维持3回合

大周天 木→火→土→金→水→木 (all T0)：
  神识/回合 = 1+1+1+1+1 = 5
  气/回合   = 1+1+1+1+1 = 5
  → 丹田=15 可维持3回合，识海=3 只能维持0回合！
  → 需要识海≥5 才能运转 → 鼓励培养识海

因果律 木→金→土→木 (all T0)：
  神识/回合 = 3+4+3 = 10
  气/回合   = 1+1+1 = 3
  → 识海消耗极高，需要大量识海容量

自环 vs 大周天：
  自环：0神识+1气/回合 — 便宜，早期可用
  大周天：5神识+5气/回合 — 强大但昂贵，后期才能持续运转
  → 自然的功力进阶曲线
```

**cost 统一带来的自然平衡：**
- 早期（丹田小）：只能维持自环几回合 → 缓慢成长
- 中期（丹田中）：可以维持简单被动 → 稳定成长
- 后期（丹田大）：可以同时维持多个被动 → 快速成长
- **丹田/识海容量 = 你的被动续航能力 = 修行的核心进度**

# 第七部分：Walk 驱动的成长系统

## 核心原则

> **所有成长来自 Walk。**
> 使用主动技能或维持被动环时，Walk 经过的节点获得灵根成长，
> 消耗的气培养丹田，消耗的神识培养识海。
> 不再有独立的"行为经验"计数器。修行就是使用 Walk 本身。

## 三维成长

每次 Walk 执行（主动技能使用一次，被动环每回合触发一次），同时产生三种成长：

| 成长目标 | 驱动维度 | 公式 |
|---------|---------|------|
| 灵根（5个节点各自） | Qi 维度 — 气如何流经节点 | 按节点贡献比例分配 qi_cost |
| 丹田容量 | Qi 维度 — 总气消耗 | `BASE_DT × qi_cost` |
| 识海容量 | ShenShi 维度 — 总神识消耗 | `BASE_SS × shenshi_cost` |

## 灵根成长公式

```
Walk 的 qi_cost = Σ(min(tier_A, tier_B) + 1)  — 即技能本身的气消耗
成长池 = BASE_LG × qi_cost  — 成长池就是气消耗的缩放

每个节点的贡献：
  weight(N) = max(0, tier(N)) + 1
    （max(0,...) 保证低于T0的节点仍有权重1，不会死锁）
  contribution(N) = weight(N) × visits(N)
    （visits = 该节点在Walk中被经过的次数）

按贡献比例分配：
  node_growth(N) = 成长池 × contribution(N) / Σ(all contributions)
```

**公式特性：**

| 特性 | 涌现方式 |
|------|---------|
| 强节点成长更快 | weight = tier+1，T3节点权重是T0的4倍 |
| 多次经过更多成长 | visits(N) 作为乘数 |
| 弱节点仍可成长 | max(0,tier)+1 保证最低权重=1 |
| 环内均衡成长 | 环中每个节点 visits=1，按 tier 比例分 |
| 同 tier 则均分 | 权重相同 → 份额相同 |
| 瓶颈效应 | edge_factor 含 min(tier) — 弱邻居降低总池 |

**计算示例（BASE_LG = 0.05）：**

```
简单技能 木(T0)→火(T0)：
  edge_factor: min(0,0)+1 = 1, pool = 0.05
  木: weight=1, visits=1, contrib=1
  火: weight=1, visits=1, contrib=1, total=2
  木 = 0.05 × 1/2 = 0.025
  火 = 0.05 × 1/2 = 0.025  ← 同tier均分

高tier技能 木(T1)→火(T2)→土(T0)：
  factors: min(1,2)+1=2, min(2,0)+1=1, pool = 0.15
  木: weight=2, 火: weight=3, 土: weight=1, total=6
  木 = 0.15 × 2/6 = 0.05
  火 = 0.15 × 3/6 = 0.075  ← 最强节点，最多成长
  土 = 0.15 × 1/6 = 0.025  ← 最弱，最少

重复经过 木(T0)→火(T0)→土(T0)→火(T0)→土(T0)→金(T0)：
  factors: 5×1=5, pool = 0.25
  木: 1×1=1, 火: 1×2=2, 土: 1×2=2, 金: 1×1=1, total=6
  木 = 0.042, 火 = 0.083, 土 = 0.083, 金 = 0.042
  ← 多次经过的节点自然获得更多

大周天被动 (全T0, 5边环, 每回合)：
  factors: 5×1=5, pool = 0.25
  各 contrib=1, total=5
  每个 = 0.25/5 = 0.05/回合
  10回合 → 每元素+0.5

大周天被动 (全T2, 5边环, 每回合)：
  factors: 5×3=15, pool = 0.75
  各 weight=3, contrib=3, total=15
  每个 = 0.75/15 × 3 = 0.15/回合 — 比T0快3倍
```

## 丹田成长公式

每次 Walk 执行时，气的总消耗培养丹田容量。

```
丹田成长 = BASE_DT × qi_cost
```

> 高 tier 节点的 Walk 消耗更多气 → 丹田成长更快。

## 识海成长公式

每次 Walk 执行时，神识的总消耗培养识海容量。

```
识海成长 = BASE_SS × shenshi_cost
  其中 shenshi_cost = Σ(distance)
```

> 高 distance 的边（侮d=3、泄d=4）消耗更多神识 → 也更能锻炼识海。
> 自环 shenshi_cost=0 → 不培养识海（无需心智引导的修行不锻炼心智）。

## 成长累积器

灵根、丹田、识海的成长为小数，但存储为整数（status turns）。
使用 PersistentVars 累积器，仅追踪队伍成员（敌人不成长）。

```lua
PersistentVars['LGACC_<guid>'] = { 木=0.3, 火=0.7, 土=0.1, 金=0, 水=0 }
PersistentVars['DTACC_<guid>'] = 0.6   -- 丹田累积器
PersistentVars['SSACC_<guid>'] = 0.4   -- 识海累积器

-- 当累积值 ≥ 1.0 时：
acc[elem] = acc[elem] - 1.0
LingGen.Add(character, elem, 1)   -- 灵根实际+1
-- 丹田/识海同理
```


## 自然循环

```
开脉 → 形成 Walk → 使用 Walk →
  流经的节点 → 灵根↑ → 边效果 tier↑ → Walk 更强
  消耗的气   → 丹田↑ → 气上限↑ → 可用更多/更强 Walk
  消耗的神识 → 识海↑ → 神识上限↑ → 可维持更多被动
  → 更多 Walk → 更多成长（三重正反馈）

策略选择：
  专精派：反复走含强元素的 Walk → 强元素快速成长 → 深度特化
  均衡派：维持大周天被动 → 五行均衡成长 → 全面发展
  识海派：走高 distance 的克/侮/泄 Walk → 识海快速成长 → 更多被动槽位
```

# 第九部分：复合系统

## 复合技能（图-图 Walk）

图本身可以作为更高层图的节点（递归组合）。

```
图 = { input: 触发元素, output: 产出元素, body: 内部结构 }
连接规则：A.output → B.input，distance 用同一公式计算
```

## 递归组合（无限层级）

```
招式      = Level 1 图  （单段 walk，战斗即时技能）
复合招式  = Level 2 图  （多段 walk 的图-图组合）
功法      = Level 3 图  （改变 walk 物理规则的修炼体系）
天道      = Level N 图  （接近宇宙规律本身）
```

## 功法：气的运行方式

```
Walk  = 路线（经过哪些节点）     ← 玩家自己可配置
功法  = 气的运行方式（物理规律） ← 功法独有，无法自配
```

### 功法能改变什么

| 功法能改变 | 自配 walk 能做到 |
|-----------|----------------|
| 节点的 distance 偏移 | ✗ |
| tier 折叠方式（min→max）| ✗ |
| 节点经过时的额外触发 | ✗ |
| final_distance 折叠方式 | ✗ |
| 解锁特殊经脉 | ✗ |
| walk 路线选择 | ✓ |

### 如何获得功法

**外得（快）：** 玉简/典籍、师传、夺取、购买
- 与金丹元素不匹配 → 效果减半；完全匹配 → 效果×1.5

**内悟（慢，完全贴合）：**
```
某条 walk 执行 ≥ N 次
+ final_distance 结果一致性 ≥ X%
+ 该 walk 的反应 tier 达到门槛
→ 功法自动从你的身体中结晶
```

## 法宝：外部子图

法宝 = 接入体内图的外部节点集合。
炼化 = 在脏器节点与法宝节点之间开通经脉。

```
简单法宝（单节点）：飞剑 = 金(T2) 节点
强力法宝（多节点）：混元鼎 = 土(T3)─火(T2)─土(T3) 内部图
至宝（完整子图）：内部有独立的5行节点 + 经脉
```

## 神识三能力

| 能力 | 效果 |
|------|------|
| **内观** | 在自身图内完整游走 → 感知所有节点状态 |
| **洞观** | 短暂外投 → 读取附近角色的图结构 |
| **掌日** | 进入目标图，强制执行 walk → 傀儡（你的神识 > 对方 → 成功） |

## 实现参考代码

```lua
ELEM_INDEX = { ["木"]=0, ["火"]=1, ["土"]=2, ["金"]=3, ["水"]=4 }

function EdgeDistance(from, to)
    if from == "丹田" then return "发元" end
    if to   == "丹田" then return "归元" end
    return (ELEM_INDEX[to] - ELEM_INDEX[from] + 5) % 5
end

function CalcWalkEffects(walk)
    local edges = {}
    local dist_sum = 0
    for i = 1, #walk - 1 do
        local d = EdgeDistance(walk[i], walk[i+1])
        local tier = math.min(GetTier(walk[i]), GetTier(walk[i+1]))
        edges[i] = { dist=d, tier=tier, effect=REACTION_TABLE[d][tier] }
        if type(d) == "number" then dist_sum = dist_sum + d end
    end
    local node_effects = {}
    for i = 2, #walk - 1 do
        local d_in  = edges[i-1].dist
        local d_out = edges[i].dist
        if type(d_in) == "number" and type(d_out) == "number" then
            local compound = (d_in + d_out) % 5
            node_effects[i] = { dist=compound, effect=REACTION_TABLE[compound][edges[i].tier] }
        end
    end
    local final_dist = dist_sum % 5
    return {
        edge_effects  = edges,
        node_effects  = node_effects,
        final_dist    = final_dist,
    }
end
```
